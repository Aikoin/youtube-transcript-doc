# 清洗 + 翻译规范

自动字幕是"毛坯"：满是语气词、口吃重复、ASR 误识。目标是产出**读起来像书面语、但仍是说话人原话**的逐字稿——清洗，不是改写，更不是摘要。

## 清洗（每一段都做）
1. **删语气词**：um、uh、作填充的 like / you know / kind of / sort of。
2. **删口吃重复**：`it it's`→`it's`、`the the`→`the`、`many many`→`many`、`should should`→`should`。
3. **补标点大小写**：让每段成为通顺的书面句子。
4. **纯语气词段整段删除**：如单独成段的 "Um,"、"Yeah."、"Y"（`parse_srt.py` 已自动删，人工复核）。
5. 保留说话人的实际用词与含义，**不要意译、不要概括**。

## 修正明显的 ASR 错误
人名、专有名词、术语最常被识别错。建立一张"术语修正表"贯穿全文替换。常见类型：
- 人名：说话人和被提及者的名字（结合视频标题/简介确认正确拼写）。
- 公司/产品名：大小写规范化（如 OpenAI、GitHub、Notion、Anthropic）。
- 技术术语：如 `Lauras`→`LoRA`、`chat GBT`→`ChatGPT`、连字符统一（`on policy`→`on-policy`）。
- 谐音误识：靠上下文判断（如某心理学家名字被拆错）。

**做法**：先通读一遍，列出该视频专属的修正表，再交给翻译子任务统一应用。

## 翻译
- 忠实、通顺的简体中文，对照清洗后的英文逐段翻译。
- 术语保留业界通用译法或原文（如 LoRA、KV cache、RAG 保留英文）。

## 分段并行
逐字稿常有近百段。可拆成 2–3 批，用并行子 agent 各译一批，**共享同一张术语修正表**保证一致，
每批输出 `{index: {en, zh}}` 的 JSON，最后合并。要求子 agent 用 `json.dump` 落盘以保证 JSON 合法。

## 版式（沉淀自真实用户反馈，重要）
这些是用户反复评论后定下的规则，务必遵守：
1. **标题保持朴素**——不要在标题里塞"中英对照""逐字稿版本"之类的强调词。
2. **不显式标 EN/ZH**——用**颜色**区分：英文行灰色（作源文参照），中文行默认黑色（主阅读）。
3. **不要整段高亮**——大段黄色底色是噪音。正文只用 **⭐** 标出高光发言段；黄色高亮留给顶部精选卡片。
   加粗 / 行内高亮是留给人工在具体短句上手动添加的，机器不要滥用。
4. **高光时刻卡片**放文档顶部，每条含 label + 一句提炼 + 可跳转时间戳。
5. 每段时间戳都是**可点击跳转**到视频对应秒的链接。

版式规则已固化进 `scripts/build_doc.py`，正常情况下直接用脚本产出即可，不用手写 XML。
