# 清洗 + 翻译规范

自动字幕是"毛坯"：满是语气词、口吃重复、ASR 误识。目标是产出**读起来像书面语、但仍是说话人原话**的逐字稿——清洗，不是改写，更不是摘要。

## 清洗（每一段都做）
1. **删语气词**：um、uh、作填充的 like / you know / kind of / sort of。
2. **删口吃重复**：`it it's`→`it's`、`the the`→`the`、`many many`→`many`、`should should`→`should`。
3. **补标点大小写**：让每段成为通顺的书面句子。
4. **纯语气词段整段删除**：如单独成段的 "Um,"、"Yeah."、"Y"（`parse_srt.py` 已自动删，人工复核）。
5. 保留说话人的实际用词与含义，**不要意译、不要概括**。

## 修正明显的 ASR 错误
人名、专有名词、术语最常被识别错。建立一张"术语修正表"贯穿全文替换。常见类型：
- 人名：说话人和被提及者的名字（结合视频标题/简介确认正确拼写）。
- 公司/产品名：大小写规范化（如 OpenAI、GitHub、Notion、Anthropic）。
- 技术术语：如 `Lauras`→`LoRA`、`chat GBT`→`ChatGPT`、连字符统一（`on policy`→`on-policy`）。
- 谐音误识：靠上下文判断（如某心理学家名字被拆错）。

**做法**：先通读一遍，列出该视频专属的修正表，再交给翻译子任务统一应用。

## 翻译
- 忠实、通顺的简体中文，对照清洗后的英文逐段翻译。
- 术语保留业界通用译法或原文（如 LoRA、KV cache、RAG 保留英文）。

## 分段并行
逐字稿常有近百段。可拆成 2–3 批，用并行子 agent 各译一批，**共享同一张术语修正表**保证一致，
每批输出 `{index: {en, zh}}` 的 JSON，最后合并。要求子 agent 用 `json.dump` 落盘以保证 JSON 合法。

## 说话人归属 + 重点标记（在清洗/翻译时一并完成）
除了 en/zh，每段还要判定这些，写进 `content.json` 的 `paras`：

1. **多说话人拆分（segs）**：ASR 常把「提问 + 回答」这类多人一来一回并进同一段。**必须**把这种合并段按语义切点拆成有序的 `segs` 列表，每个 seg 是一个子发言 `{speaker, en, zh}`；en 与 zh 的切点要 1:1 对应（双列卡两边段数一致）。单人连续段就只有 1 个 seg。

2. **说话人名字：不确定就别编（铁律，踩过坑）**。
   - **只有当字幕本身带说话人标签、或你非常确定时**，才在 seg 里填 `speaker` 名字（渲染成加粗 `名字：` 前缀）。
   - **大多数情况：SRT 只有 `>>` 换人标记、没有名字**——「谁说的」纯属推断，容易张冠李戴。这种情况 `speaker` **一律留空 `""`**，不要硬安名字。build_doc 会在每个「换到的」子段前渲染一个中性加粗破折号 `— `，让读者看出「换人了」，但不武断断言是谁。
   - 判据：`>>` 后面直接是台词而非 `Name:`，就是无名字字幕 → 走留空 + 破折号方案。

3. **key（是否重点）**：`true` 给真正给出答案 / 抛出核心观点的实质性回合；`false` 给铺垫、提问、寒暄的回合。
   `key` 只用于挑选做节 summary 的素材，**正文不据此整段加粗**（整段加粗=噪音）；重点强调由节标题下的 💡「重点」卡片承担。
4. **每节 summary**：给每个 `section` 写一句话「重点」概述（不是搬原句，是提炼），渲染成节标题下的 💡 卡片。

## 版式（沉淀自真实用户反馈 + 对标 codex 更清爽的产出，重要）
这些是用户反复评论后定下的规则，务必遵守：
1. **标题保持朴素**——不要在标题里塞"中英对照""逐字稿版本"之类的强调词。
2. **正文用双列 callout 卡片**——每个 turn 一张两列网格卡片：左列英文、右列中文，比一堵 `<p>` 墙更好扫读。
3. **靠「卡片颜色」区分角色，不靠加粗、不显式标 EN/ZH**（踩过坑的关键）：左列英文 = 中性 **light-gray** 卡（💬，正文灰字，参考/原文列）；右列中文 = 主色 **light-blue** 卡（💬，正文黑字，主读列）；节「重点」= **light-green**（💡）；顶部高光 = **light-yellow**（⭐）。四色四角色，一眼分清——**严禁所有卡片同一背景色**（用户否掉过"同一 callout 背景、同一字色、满屏加粗"的杂乱版）。
4. **卡内多说话人用 segs 分段，换人处标记而非硬安名字**——一张卡里每个子发言单独一段 `<p>`；换到的子段前加一个加粗前缀：**有确定名字**就标 `名字：`，**没名字**（`>>` 无标签字幕）就用中性破折号 `— `，绝不编造说话人。时间戳 + ⭐ 单独成头部一行。
5. **绝不整段加粗**——卡片正文任何时候都不整段加粗（整段加粗=噪音）；加粗只留给说话人名字/换人破折号前缀。`key` 的重点强调交给节标题下的 💡 绿色「重点」卡片。
6. **每节一张 💡 绿色「重点」卡片**——节标题下先给一句话主旨 + 可点击时间戳。
7. **高光时刻卡片**放文档顶部（黄色），每条含 label + 一句提炼 + 可跳转时间戳；正文里对应的高光段只在卡片头部加 **⭐** 标记。
8. 每段时间戳都是**可点击跳转**到视频对应秒的链接。

版式规则已固化进 `scripts/build_doc.py`，正常情况下直接用脚本产出即可，不用手写 XML。
