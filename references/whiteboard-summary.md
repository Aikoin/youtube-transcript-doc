# 第 6 步（必做）：生成可视化总结白板

发布逐字稿后**必须**再产出一张**提炼重点的可视化总结白板**，嵌进同一篇文档。这是标准交付物，不是可选项。

**画图直接交给 `beautiful-feishu-whiteboard` skill**（本环境已安装的专业绘图引擎，用 Skill 工具启动）。它内置了完整的构图模式选择、视觉节奏、跨区域映射、SVG 编码到画板上传的工作流，比自己手搓排版靠谱得多。本 skill 只负责：**提炼结构 + 指定目标文档**，把这两样交过去。

## 什么时候做
逐字稿发布后，每次都做（不再等用户开口）。

## 流程

### 6.1 把访谈提炼成结构（本 skill 做）
从 `content.json` / turns 里抽出 4–8 个**主题簇**，每簇 3–5 条要点。别把逐字稿原句搬上去，要提炼成短句。
尽量提炼出**关系**（A 依赖 B、A 产出 B、X 对应 Y），因为 `beautiful-feishu-whiteboard` 的核心就是把关系画成「关系地图」而非「PPT 罗列」。

### 6.2 交给 beautiful-feishu-whiteboard skill 画（用 Skill 工具启动）
把 6.1 提炼的结构 + 已发布逐字稿文档的 `document_id` 交给它，明确要求：
- **嵌进本篇逐字稿文档**，而不是新建一篇文档；
- **位置铁律：白板放在「⭐ 高光时刻」和「📝 完整逐字稿」之间**，不是文档尾部。做法：`docs +fetch` 拉出块结构，找到高光区结尾的那条分割线块，用 `docs +update --command block_insert_after --block-id <分割线块>` 插入 `<h1>🗺️ 核心观点可视化总结</h1><whiteboard type="blank"></whiteboard>`，拿到新块 `block_token` 再写图。（如果先前误插到尾部，用 `block_delete` 删掉旧的那块和它的标题。）
- 内容是这期视频的核心观点总结，主题簇之间尽量画出关系连线；
- 走完它自己的「渲染 → 检查 → 写入画板 → 回查线上真图」全流程，最终把白板落到文档里。

构图模式、配色体系、形状规范、映射线技法、写入画板与质量自检等细节，全部由 `beautiful-feishu-whiteboard` 的 SKILL.md 负责，这里不重复。

## 仍要盯住的两条踩坑（转达给绘图 skill / 自检时确认）
飞书白板**渲染时会把文字强制压成深色**，所以：
- **所有文字都放在浅色底（奶油/白/浅色）上用深墨色**，保证对比度；彩色只用于边框、标题条、色块标记、徽章、连线，**绝不把文字放到饱和色块上**；
- **别信任本地 PNG 预览的文字颜色**——它和线上不一致。必须写入后回查**线上真图**再确认对比度，有问题再改。

（这条是踩坑沉淀：曾把标题文字设成浅色放在绿/粉色条上，线上渲染全变深色压深底、几乎看不清，返工才发现。）
